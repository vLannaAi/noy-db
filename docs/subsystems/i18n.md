# i18n

> **Subpath:** `@noy-db/hub/i18n`
> **Factory:** `withI18n()`
> **Cluster:** C — Data Shape
> **LOC cost:** ~854 (off-bundle when not opted in)

## What it does

Multi-locale records via two complementary patterns: `dictKey` (stable enum-like keys → shared label dictionary) and `i18nText` (per-record free-form text in multiple locales). Locale resolution happens at read time. Dictionary itself is encrypted and versioned.

## When you need it

- Apps that ship in more than one language
- Enum-like fields (`status`, `taxClass`, `country`) where labels differ by locale
- Per-record translatable content (descriptions, comments, free-form notes)
- Auto-translate hooks for missing locale fills

## Opt-in

```ts
import { createNoydb, dictKey, i18nText } from '@noy-db/hub'
import { withI18n } from '@noy-db/hub/i18n'

const db = await createNoydb({
  store: ...,
  user: ...,
  i18nStrategy: withI18n(),
})

const vault = await db.openVault('firm')
await vault.dictionary('status').putAll({
  draft: { en: 'Draft', th: 'ฉบับร่าง' },
  paid:  { en: 'Paid',  th: 'ชำระแล้ว' },
})

vault.collection<Invoice>('invoices', {
  dictKeyFields: {
    status: dictKey('status', ['draft', 'paid'] as const),
  },
  i18nFields: {
    description: i18nText({ languages: ['en', 'th'], required: 'all' }),
  },
})
```

## API

- `vault.dictionary(name).putAll(...)` / `put(...)` / `get(...)` / `delete(...)` / `list()` / `rename(...)`
- `dictKey(name, keys)` and `i18nText({ languages, required })` field descriptors
- `resolveI18nText(value, locale, fallback?)` helper — resolve a single value
- Per-call `{ locale, fallback }` on `collection.get()` / `list()` / `query()` reads

## Behavior when NOT opted in

- `applyI18nLocale` (under the hood) returns the record unchanged — locale resolution is silently skipped
- `validateI18nTextValue` throws when a collection declared `i18nFields` (misconfiguration: declared but didn't opt in)
- `vault.dictionary(name)` throws with a pointer to `@noy-db/hub/i18n`

## Pairs well with

- **aggregate** — `groupBy(dictKeyField)` resolves labels in the result on render
- **history** — locale changes are versioned

## Edge cases & limits

- Required locales declared in `i18nText({ required: 'all' })` are enforced on put — missing translations throw `MissingTranslationError`
- `'auto'` translate fields call out to the `plaintextTranslator` callback set at `createNoydb`-time
- Dictionary `rename()` rewrites every collection record that points at the old key — O(records) per rename
