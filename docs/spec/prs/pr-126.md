## Summary

Implements five v0.7 features in `@noy-db/core`. Zero new npm dependencies — all crypto via Web Crypto API.

- **Session tokens (#109)** — non-extractable, tab-scoped AES-256-GCM session key wraps the serialised DEK map as a `SessionToken`. Closing the tab destroys the key and revokes the session. New: `createSession`, `resolveSession`, `revokeSession`, `revokeAllSessions`, `isSessionAlive`, `activeSessionCount`, `SessionExpiredError`, `SessionNotFoundError`.

- **Session policies (#114)** — idle / absolute timeouts, `requireReAuthFor` operation guard, `lockOnBackground` tab-visibility hook wired into `Noydb`. New: `PolicyEnforcer`, `createEnforcer`, `validateSessionPolicy`, `SessionPolicy` type, `ReAuthOperation` type, `SessionPolicyError`.

- **`_sync_credentials` reserved collection (#110)** — ACL-gated (owner/admin only) encrypted per-adapter OAuth token store, parallel to `_keyring`/`_ledger`. Adapters only see ciphertext. New: `putCredential`, `getCredential`, `deleteCredential`, `listCredentials`, `credentialStatus`, `SYNC_CREDENTIALS_COLLECTION`.

- **Magic-link unlock (#113)** — `HKDF-SHA256(serverSecret, SHA256(token), "noydb-magic-link-v1:"+compartment)` derives a viewer KEK for one-shot read-only client-portal sessions. Compartment-bound: a token for compartment A cannot unlock B. New: `deriveMagicLinkKEK`, `createMagicLinkToken`, `isMagicLinkValid`, `buildMagicLinkKeyring`.

- **Dev-mode persistent unlock (#119)** — opt-in escape hatch for hot-reload inner-loop friction. Stores DEKs in `sessionStorage` / `localStorage` with hard guards: `NODE_ENV=production` throws, non-localhost throws, `acknowledge: 'I-UNDERSTAND-THIS-DISABLES-UNLOCK-SECURITY'` required, unsuppressable `console.warn`. New: `enableDevUnlock`, `loadDevUnlock`, `clearDevUnlock`, `isDevUnlockActive`.

Also in this PR:
- `scripts/release.mjs` + `pnpm release:version` — version normalizer that prevents the changeset 0.x→1.0 major-bump heuristic (documented in `docs/v0.6/retrospective.md`)
- `docs/v0.7/merge-runbook.md`
- `bufferToBase64`, `base64ToBuffer`, `UnlockedKeyring` exported from core (needed by auth packages)
- `base64ToBuffer` return type narrowed to `Uint8Array<ArrayBuffer>` (TypeScript 5.9 strictness)
