### Updated

- **\`CONTRIBUTING.md\`** — encodes the three lessons the v0.4 epic taught us:

  1. **New section 7a: the non-negotiable pre-publish smoke test.** Every release since v0.3.1 has been bitten by a bug that only shows up when you install packed tarballs in a fresh directory. Workspace symlinks hide three separate classes of bug (runtime deps as devDeps, \`workspace:*\` in peer deps, missing files in the package's \`files\` list). Now documented with a concrete pack-and-smoke-test recipe.

  2. **New section 7b: \`workspace:*\` vs \`workspace:^\` for peer deps.** Critical distinction: \`workspace:*\` in \`peerDependencies\` publishes as an exact version pin, breaking cross-version installs. Use \`workspace:^\` for peer deps unconditionally. This is the v0.4.0 → v0.4.1 bug codified.

  3. **New section 8: starting a new release epic.** Checklist for tracking issue + sub-issues + milestone + long-lived branch + branch protection. The "create a milestone alongside the epic" step was missed for v0.4 and created retroactively — now documented so v0.5+ gets it right from day one.

- **\`README.md\`** — status table explicitly lists v0.4.1 (was only showing v0.4). Header line now says "v0.4.1 shipped on npm" with a note about the unified version line.
