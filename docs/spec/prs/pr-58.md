## New sections (4)

### 1. Why noy-db exists

A ~150-word framing that inverts the default assumption of data-storage libraries. Most tools assume you'll rent a database from someone; noy-db assumes your data belongs on your own devices. Closes with: _\"Data ownership isn't a feature we shipped. It's the shape of the whole project.\"_

### 2. Key factors (5 bullets)

- 🔒 **Zero-knowledge encryption** — AES-256-GCM, per-user keys, adapters only see ciphertext
- ☁️ **Serverless by design** — **no noy-db server to run**. No Docker, no managed service, no backend. The library embeds in your app; you own the data end-to-end.
- 📱 **Runs on any OS, any device** — macOS/Linux/Windows/iOS/Android/Raspberry Pi/Node/Bun/Deno/browser. **Minimum requirements: a JS engine and the Web Crypto API.**
- 🌐 **Offline-first** — every operation works without internet. Sync is opportunistic.
- 👥 **Multi-user built in** — 5 roles, per-collection permissions, no auth server required.
