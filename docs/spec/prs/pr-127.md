## Summary

New package `@noy-db/auth-webauthn`. Enables hardware-key keyrings using the WebAuthn API — Touch ID, Face ID, Windows Hello, YubiKey, SoloKey, any FIDO2 key.

- **PRF extension** for deterministic key derivation (output is authenticator-bound; same device/credential always produces the same key). `rawId`-HKDF fallback for authenticators without PRF support.
- **BE-flag guard** — reads authenticator data byte 32 bit 3. If `requireSingleDevice: true` and the credential is backup-eligible (synced to iCloud Keychain, Google Password Manager, etc.), throws `WebAuthnMultiDeviceError` at both enrollment and assertion time.
- Full `enrollWebAuthn` → `unlockWebAuthn` round-trip for both PRF and rawId paths.

Peer dep: `@noy-db/core` (workspace:*). Zero additional npm deps.

> **Note:** depends on the new `bufferToBase64`/`base64ToBuffer`/`UnlockedKeyring` exports added in PR #126. Merge #126 first.
