## Summary

Three new \`noy-db\` subcommands for routine key management and backup, plus a new \`Noydb.rotate()\` core method. Closes #38 (deferred from the v0.3.1 scope reduction).

## New subcommands

| Command | Purpose |
|---|---|
| \`noy-db rotate\` | Rotate DEKs for one or more collections. Everyone keeps access with fresh keys. |
| \`noy-db add user <id> <role>\` | Grant a new user access to a compartment. Two passphrase prompts (caller + new, with confirmation). |
| \`noy-db backup <target>\` | Dump a compartment to a local file. \`file://\` + plain paths supported. |

All three use the file adapter, require \`--dir --compartment --user\` flags, and prompt for the passphrase via clack's \`password()\` (never echoed, never logged).

## Core addition

New **\`Noydb.rotate(compartment, collections[])\`** method. Previously rotation was only accessible bundled with \`revoke({ rotateKeys: true })\`, which also kicks a user out. The new method rotates key material without removing anyone — the CLI \`rotate\` subcommand uses it.

## Security invariants every subcommand enforces

1. Passphrase read via \`@clack/prompts\` \`password()\` → never echoes
2. Passphrase never logged, never in error messages, never beyond the local closure
3. \`finally { db.close() }\` clears the KEK from memory on exit (success or failure)
4. Ctrl-C at the password prompt aborts **before** any I/O happens
5. Unsupported backup schemes (\`s3://\`, \`https://\`) rejected **before** the passphrase prompt — typos don't waste a prompt entry

## Testability via dependency injection

Each subcommand takes three optional injected deps:

\`\`\`ts
interface RotateOptions {
  dir: string
  compartment: string
  user: string
  collections?: string[]
  readPassphrase?: ReadPassphrase   // default: clack prompts
  createDb?: typeof createNoydb      // default: real factory
  buildAdapter?: (dir) => Adapter    // default: jsonFile
}
\`\`\`

Production code leaves all three undefined. Tests pass scripted stubs so they run in ~4s total without touching stdin, without spawning subprocesses, and without hitting disk (except \`backup\`, which writes to \`os.tmpdir()\` and cleans up in \`afterEach\`).
