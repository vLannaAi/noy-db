### Manifest-only packages (republished at 0.5.0 for peer dep refresh)

\`@noy-db/browser\`, \`@noy-db/dynamo\`, \`@noy-db/s3\`, \`@noy-db/pinia\`, \`@noy-db/vue\`, \`@noy-db/nuxt\` — no functional changes, but republished so that their peer dependencies expand to \`^0.5.0\` instead of \`^0.4.1\`. In 0.x semver, caret ranges are locked to the minor (\`^0.4.1\` = \`>=0.4.1 <0.5.0\`), so without the unified line bump, installing one of these at 0.4.1 alongside \`@noy-db/core@0.5.0\` would hit \`ERESOLVE\`. This is the same pattern established in v0.4.1.

### Breaking changes

- **\`Compartment.export()\` removed** in \`@noy-db/core\`. One-line migration: \`comp.export()\` → \`comp.exportJSON()\`. The new method is strictly more capable (ACL-scoped instead of owner-only, schema + refs metadata surfaced, streaming output).
