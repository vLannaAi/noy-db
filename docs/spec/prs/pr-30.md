## Bonus bug fix in \`.changeset/config.json\`

\`linked: [["@noy-db/*"]]\` was nonsense — the changesets \`linked\` field expects exact package names, not glob patterns. The misconfigured glob caused \`pnpm changeset version\` to produce \`2.0.0\` bumps instead of \`0.3.0\` (manually reproduced and reverted in this branch). Replaced with \`linked: []\` so future releases version each package independently against its own changesets.

I generated the CHANGELOG entries by hand from the existing v0.3 changeset content rather than fight the broken CLI behavior — the resulting changelogs are richer than what \`@changesets/cli/changelog\` would have produced anyway, since they include cross-references and benchmark numbers.
