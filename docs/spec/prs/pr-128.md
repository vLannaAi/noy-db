## Summary

New package `@noy-db/auth-oidc`. Enables federated login (LINE, Google, Apple, Okta, Keycloak, any OIDC provider) without the key-connector server ever seeing the plaintext KEK.

**Split-key model (Bitwarden-style key connector):**
```
serverHalf ⊕ deviceHalf = KEK
```
- `deviceHalf` derived via HKDF(deviceSecret, "noydb-oidc-device-v1:"+compartment). Device-local, never transmitted.
- `serverHalf = KEK ⊕ deviceHalf`. Encrypted with `HKDF(idToken)` before transmission.
- Key-connector server stores the encrypted `serverHalf` indexed by OIDC `sub`.
- At unlock: client fetches `serverHalf`, derives `deviceHalf`, XORs to reconstruct KEK.

**Threat model summary:**
- Stolen OIDC token only → attacker has `serverHalf` but not `deviceHalf`. KEK not reconstructable.
- Compromised key-connector only → attacker has all `serverHalf`s but not `deviceHalf`s. KEK not reconstructable.
- Both required simultaneously for full compromise.

**Built-in provider helpers:** `knownProviders.line()`, `knownProviders.google()`, `knownProviders.apple()`.

Peer dep: `@noy-db/core` (workspace:*). Zero additional npm deps.
