## Summary

- The `@noy-db/create` wizard now speaks **Thai** in addition to English
- Pick a language with `--lang en|th`, or let the wizard auto-detect from POSIX `LC_ALL` / `LC_MESSAGES` / `LANG` / `LANGUAGE`
- Validation errors and stack traces stay in English so bug reports look the same across locales

## What's translated

Every user-facing prompt, note, and confirmation in both fresh-mode and augment-mode flows. Adapter labels are translated too.

## What's NOT translated (deliberate)

- Validation errors / stack traces — keeps issue tracker triage simple regardless of who files the bug
- Generated project source code — translating TS templates would just create a fork
- Technical identifiers (`modules`, `noydb:`, framework names, adapter names) — that's how Thai devs actually write code

## Architecture

- New `packages/create-noy-db/src/wizard/i18n/` with `types.ts`, `en.ts`, `th.ts`, and `index.ts`
- 26-key `WizardMessages` interface enforces compile-time shape conformance
- `detectLocale(env)` follows POSIX precedence (`LC_ALL` > `LC_MESSAGES` > `LANG` > `LANGUAGE`), strips encoding/modifier/region (`th_TH.UTF-8@euro` → `th`), falls back to `en` for unsupported locales
- Sync `import` of bundles (not dynamic) — they're tiny and async would force every prompt callsite to be async
- Public API re-exported from package root: `detectLocale`, `loadMessages`, `parseLocaleFlag`, `SUPPORTED_LOCALES`, `Locale`, `WizardMessages`

## Tests
