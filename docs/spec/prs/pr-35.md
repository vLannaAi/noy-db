## Summary

**Patch release fix.** v0.3.1 shipped with a broken \`noy-db verify\` command because \`@noy-db/core\` and \`@noy-db/memory\` were declared as \`devDependencies\` instead of \`dependencies\`. The wizard (\`create\` bin) worked fine because it only writes files, but \`pnpm exec noy-db verify\` threw \`ERR_MODULE_NOT_FOUND\` as soon as a user installed \`@noy-db/create\` from npm.

Found while dogfooding the v0.3.1 release against a fresh \`npm install\` in a temp dir.

## Fix

Move both packages to \`dependencies\` so they're resolved from \`node_modules/@noy-db/create/node_modules/\` at runtime. No API or code changes beyond the manifest.
