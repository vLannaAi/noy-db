Spawned from discussion vLannaAi/noy-db#78 — i18n as a first-class primitive.

### NOYDB_SPEC.md — sharpens the zero-knowledge invariant

The spec previously said "backends never see plaintext" but never explicitly addressed what zero-knowledge does **not** promise. This PR adds a new subsection under §Design Principles → Zero-Knowledge Storage that:

- Scopes the guarantee to **adapters specifically** — no plaintext, no key, no decrypted DEK is ever passed to an adapter method
- Explicitly states the guarantee is **not** that plaintext never leaves the process
- Names the three deliberate, documented mechanisms by which plaintext exits: `@noy-db/decrypt-*` packages, the v0.8 `plaintextTranslator` hook, and consumer-supplied schema validators
- Documents that audit ledger entries for these exits record metadata only — **never** content, **never** content hashes (the hash would be a fingerprint that allows correlation of identical phrases)
- Documents the discipline a consumer can apply to keep plaintext in process memory only

This clarification is **independently useful** — it sharpens a point that was underspecified in the spec regardless of whether i18n ships.

### ROADMAP.md — new v0.8 "Internationalization & dictionaries" milestone

- **`dictKey('name')`** — schema type for bounded enum-like fields backed by an encrypted per-compartment dictionary collection. One collection per dictionary (composes with v0.4 refs); type-narrowed via `as const` keys at schema-construction time; `groupBy(dictKey)` groups by stable key (type-enforced); `dictionary.rename()` replaces dangerous cascade-on-delete
- **`i18nText({ languages, required })`** — schema type for per-record multi-language prose with declarative locale fallback chain and raw mode for bilingual exports
- **`plaintextTranslator` hook** — consumer-supplied, library ships no translator and no SDK, named "plaintext" deliberately to make the boundary crossing visible at every read of the config
- **Export integration** — `exportStream()` (#72) carries a bundled dictionary snapshot so exports remain internally consistent forever
- **Out of scope, named explicitly:** pluralization, date/number/currency formatting, RTL/LTR rendering, per-locale CRDT merging (deferred to v0.9 sync v2), codegen for type narrowing, cross-compartment shared dictionaries
